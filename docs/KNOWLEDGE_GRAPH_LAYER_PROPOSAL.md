# 知识图谱(KG)构建工具层级选择方案

**提议者**: Six-in-One Hub  
**讨论者**: Chief Architect + Knowledge Guardian  
**日期**: 2026-05-03  
**目标**: 确定知识图谱构建工具在A5L五层架构中的位置

---

## 🎯 工具定位

**工具名称**: `knowledge_graph_builder.py` / `kg_visualizer.py`  
**核心功能**:
- 从归档的研报/文档中提取实体（股票、行业、概念、事件）
- 建立实体间关系（产业链、竞争、关联、因果）
- 可视化展示知识图谱
- 支持投资逻辑推导

---

## 🏛️ 层级选择分析

### 选项1: Layer1 - Data Foundation（数据底座层）

**理由**:
- 知识图谱是结构化的数据资产
- 为上层分析提供关联数据支持
- 类似"知识数据库"角色

**优点**:
- 与现有数据工具（统一股票价格、新闻聚合）同层
- 数据层定位清晰
- 便于其他层调用

**缺点**:
- 知识图谱不仅是数据，还包含"智慧"
- 可能与Layer1纯数据定位不符

**适合度**: ⭐⭐⭐☆☆

---

### 选项2: Layer2 - Strategy Engine（交易策略层）⭐ 推荐

**理由**:
- 知识图谱直接服务于投资决策
- 产业链关联、竞争关系是选股依据
- 与因子投资、量化分析等策略工具同层

**优点**:
- 投资逻辑链：图谱关联 → 发现机会 → 生成策略
- 与现有策略工具形成互补
- 策略层需要"知识驱动"能力

**缺点**:
- 知识图谱也服务于分析层

**适合度**: ⭐⭐⭐⭐⭐

**推荐位置**: `TOOLS/knowledge_graph/` 或 `skills/knowledge-graph/`

---

### 选项3: Layer3 - Unstructured Analysis（非结构化分析层）

**理由**:
- 知识图谱是A5L智慧的沉淀
- 与UZI分析、产业链分析同层
- 是非结构化数据的结构化输出

**优点**:
- 与分析层"智慧"定位一致
- 可直接被A5L调用分析
- 是分析成果的积累

**缺点**:
- 分析层偏重"分析过程"，知识图谱偏重"结果沉淀"
- 可能与分析层工具职责有重叠

**适合度**: ⭐⭐⭐⭐☆

---

### 选项4: Layer4 - Decision Signal（决策信号层）

**理由**:
- 知识图谱可直接产生交易信号
- 事件关联 → 信号生成

**优点**:
- 直接服务于交易决策

**缺点**:
- 知识图谱不是直接产生信号，而是支持信号生成
- 定位过于下游，限制了应用范围

**适合度**: ⭐⭐☆☆☆

---

### 选项5: Layer5 - Review & Evolution（复盘进化层）

**理由**:
- 知识图谱用于复盘投资决策
- 追踪投资逻辑演变

**优点**:
- 复盘时可以查看知识关联

**缺点**:
- 知识图谱是实时构建的，不只是复盘使用
- 定位过于后期

**适合度**: ⭐⭐☆☆☆

---

## 💡 Six-in-One Hub 建议

**首选**: **Layer2 - Strategy Engine（交易策略层）**

**理由**:
1. 知识图谱是"投资策略的基础设施"
2. 与因子投资、量化分析等工具形成互补
3. 策略层需要知识驱动的能力
4. 既服务于投资决策，又可被分析层调用

**次选**: **Layer3 - Unstructured Analysis（非结构化分析层）**

**理由**:
- 如果强调"知识是A5L智慧的结晶"，可放在分析层
- 但分析层更侧重"分析过程"而非"知识沉淀"

## 🔧 技术实现建议

### 如果放在Layer2

```
skills/
  knowledge-graph/
    SKILL.md              # 知识图谱构建SKILL
    knowledge_graph_builder.py  # 构建引擎
    kg_visualizer.py      # 可视化
    data/                 # 图谱数据存储
      entities.json       # 实体库
      relations.json      # 关系库
      graph.db            # 图数据库
```

### 核心功能模块

1. **实体提取器**
   - 从研报提取：股票、行业、概念、事件、人物
   - 实体消歧：同名不同实体识别

2. **关系构建器**
   - 产业链关系：上游/下游/竞争
   - 关联关系：同行业、同概念
   - 因果关系：事件→影响

3. **图谱可视化**
   - 交互式图谱展示
   - 路径分析：发现隐藏关联
   - 热点图谱：当前市场热点

4. **查询接口**
   - 供其他SKILL调用
   - API: `get_related_stocks(stock)`, `get_industry_chain(industry)`
