## 📝 交易计划模板（CIO标准格式）

### 计划必须包含的7要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **1. 标的** | 股票代码+名称 | 致尚科技(301486.SZ) |
| **2. 方向** | 买入/卖出/持有 | 买入 |
| **3. 逻辑** | 2-4点关键因素 | ①Senko Q3量产 ②价值量3-5倍 ③龙头采用 |
| **4. 仓位** | 占总资金比例 | 5%初试，验证后加至10% |
| **5. 时机** | 触发条件 | 开盘30分钟内观察量能，确认后下单 |
| **6. 风控** | 止损/止盈/时间止损 | 技术止损-8%，时间止损Q3未量产减仓 |
| **7. 验证** | 模拟盘验证天数 | 3-5天模拟验证后再考虑实盘 |

## 🔄 复盘模板（三层复盘）

### 第一层：日度复盘（Daily Review）
- **计划执行情况**：是否按计划执行？偏差原因？
- **市场验证**：预期vs实际，逻辑是否被验证？
- **盈亏归因**：α收益（逻辑正确）vs β收益（市场波动）
- **SKILL更新**：新发现的方法论/错误模式

### 第二层：周度复盘（Weekly Review）
- **胜率统计**：计划准确率、盈亏比
- **策略有效性**：哪些策略有效？哪些失效？
- **市场理解**：宏观环境变化对策略的影响
- **下周计划**：基于复盘的策略调整

### 第三层：月度复盘（Monthly Review）
- **整体收益率**：vs 基准指数
- **最大回撤**：风控效果评估
- **SKILL进化**：方法论升级记录
- **系统优化**：工具/流程改进建议

## 🎮 模拟→实盘升级机制

```
模拟盘验证 → 连续3次计划准确 → 小仓位实盘(5%) 
                                         ↓
     复盘优化 ← 验证成功 ← 连续盈利 ← 加仓(10%)
          ↓
     形成标准化SKILL → 全市场复制
```

### 升级条件

| 阶段 | 条件 | 仓位 |
|------|------|------|
| **模拟验证** | 计划准确率>70%，连续3次 | 0%（纯模拟） |
| **小仓位实盘** | 模拟验证通过 | 5% |
| **标准仓位** | 小仓位连续盈利 | 10% |
| **重仓** | 策略成熟，高确定性 | 15-20% |
