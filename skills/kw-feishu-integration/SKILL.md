# KW-Feishu Integration - 知识联动系统

> **版本**: v1.0.0  
> **创建时间**: 2026-05-08  
> **目标**: 解决KW与飞书云文档割裂问题，建立深度联动

**描述**: KW-Feishu Integration Engine实现本地Karpathy Wiki与飞书云文档的双向深度联动。核心能力：(1) KW→飞书：自动推送成熟知识到飞书归档；(2) 飞书→KW：提取飞书文档关键insight回流本地；(3) 统一索引：跨平台一站式检索；(4) 链接映射：建立双向引用关系。实现"本地加工+云端归档"的知识管理闭环。

---

## 🎯 核心问题与解决方案

### 原有问题

| 问题 | 现象 | 解决方案 |
|:-----|:-----|:---------|
| **割裂** | KW和飞书各管各 | **角色分离**: KW加工车间，飞书档案馆 |
| **不同步** | KW更新了飞书不知道 | **自动推送**: 成熟知识自动同步到飞书 |
| **无回流** | 飞书读完insight丢失 | **提取引擎**: 标记段落自动回流KW |
| **难检索** | 不知道知识在哪 | **统一索引**: 一次搜索跨平台 |

---

## 🏗️ 三层联动架构

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: 原始信息层                                        │
│  ├── 研报PDF → KG归档 → sources/                           │
│  ├── 聊天记录 → KG整理 → sources/                          │
│  └── 飞书文档 → API拉取 → sources/feishu/                  │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 知识编译层 - KW主场                               │
│  ├── 实体提取 (公司/人物/概念/事件)                         │
│  ├── 深度分析 (CTF/UZI/产业链)                             │
│  ├── 关系映射 ([[双向链接]])                               │
│  └── 生成wiki页面 (本地知识网络)                           │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: 云端归档层 - 飞书主场                             │
│  ├── KW成熟知识 → 自动推送 → 飞书                          │
│  ├── 飞书文档 → 标记提取 → 回流KW                          │
│  └── 统一索引 (本地+云端一站式检索)                        │
└─────────────────────────────────────────────────────────────┘
```

## 🔄 双向联动工作流

### 工作流A: KW → 飞书 (正向归档)

**触发时机**: 每日收盘后 / 知识成熟时

```
KG归档今日信息
    ↓
KW编译深度分析
    ├── [[公司/中国长城]] - Tier 1分析
    ├── [[事件/2026-05-07_4连板]] - 催化验证
    └── 关联: [[概念/CPU服务器大周期]]
    ↓
KW-Feishu引擎自动推送
    ├── 标题: "个股分析_20260507_中国长城(000066)"
    ├── 位置: 飞书/空间2/20_个股档案/
    └── 格式: 飞书兼容Markdown
    ↓
飞书API自动创建文档
    ↓
Chief收到通知: "中国长城分析已归档至飞书"
```

**归档规则**:

| KW页面类型 | 飞书目标位置 | 命名规范 |
|:-----------|:-------------|:---------|
| wiki/companies/ | 空间2/20_个股档案/ | 个股分析_YYYYMMDD_名称(代码) |
| wiki/events/ | 空间2/30_每日批注/ | 事件分析_YYYYMMDD_事件名 |
| wiki/concepts/ | 空间2/10_行业研究/ | 概念研究_YYYYMMDD_概念名 |

---

### 工作流B: 飞书 → KW (反向回流)

**触发时机**: Chief在飞书阅读时标记关键段落

```
Chief阅读飞书研报
    ↓
高亮关键段落 (使用标记)
    ├── 💡 Insight: "AMD TAM翻倍"
    └── 📌 重点: "建议关注国产CPU"
    ↓
KG捕获高亮 (定时扫描)
    ↓
提取到KW sources/
    ├── feishu_extract_研报名称.json
    └── 包含: 原文+来源+时间戳
    ↓
KW编译时自动关联
    ├── 创建 [[事件/AMD_TAM翻倍]]
    ├── 关联 [[公司/中国长城]]
    └── 引用来源: [飞书文档](feishu://...)
    ↓
更新知识网络
```

**飞书标记规范**:

| 标记 | 含义 | KW处理 |
|:-----|:-----|:-------|
| 💡 | 核心Insight | 创建概念/事件页面 |
| 📌 | 重要数据 | 更新公司基本面 |
| ❓ | 待验证问题 | 创建待办任务 |
| 🔗 | 关联引用 | 建立双向链接 |

## 📋 角色分离原则

| 维度 | KW (本地) | 飞书 (云端) |
|:-----|:----------|:------------|
| **核心定位** | 知识加工车间 | 知识档案馆 |
| **内容状态** | 半成品+深度分析 | 成品+官方归档 |
| **更新频率** | 每日实时 | 周度/月度批量 |
| **使用场景** | 快速查询+交叉引用 | 对外分享+长期存储 |
| **存什么** | 关联网络+推理过程 | 最终结论+执行记录 |
| **检索方式** | [[双向链接]]浏览 | 飞书搜索+目录导航 |

## 🚀 使用方法

### 1. KW → 飞书 同步

```bash
# 同步单个公司页面
python3 TOOLS/kw_feishu_engine.py sync wiki/companies/中国长城.md "20_个股档案"

# 同步所有公司页面 (批量归档)
python3 TOOLS/kw_feishu_engine.py sync-all
```

### 2. 飞书 → KW 提取

```bash
# 从飞书文档提取关键insight
python3 TOOLS/kw_feishu_engine.py extract /path/to/研报.md
```

### 3. 统一搜索

```bash
# 跨平台搜索知识
python3 TOOLS/kw_feishu_engine.py search "中国长城"
# 输出:
#   本地KW: 3条 (公司页面/事件页面/概念页面)
#   飞书: 2条 (个股分析_20260507/个股分析_20260508)
```

### 4. 查看链接映射

```bash
python3 TOOLS/kw_feishu_engine.py map
# 输出:
#   wiki/companies/中国长城.md → 空间2/20_个股档案/个股分析_20260507_中国长城
#   wiki/companies/盈峰环境.md → 空间2/20_个股档案/个股分析_20260507_盈峰环境
```
