# Hermes - 统一消息协调中心 (Unified Messaging Hub)

> **版本**: v1.0.0  
> **代号**: Hermes  
> **定位**: A5L Layer 0 通信层 - 所有消息的单一出口  
> **创建时间**: 2026-05-08

**描述**: Hermes是A5L系统的统一消息协调中心，整合所有通知渠道(飞书、系统提示、定时提醒等)为单一出口。作为"消息的中转站"，Hermes负责消息路由、优先级排序、去重合并、定时发送，确保Chief在正确的时间收到正确的信息，避免消息轰炸和遗漏。

---

## 🎯 核心问题

### 原有问题

| 问题 | 现象 | 影响 |
|:-----|:-----|:-----|
| **消息碎片化** | 多个系统各自发消息 | Chief被消息轰炸 |
| **优先级混乱** | 重要消息淹没在通知中 | 关键信息遗漏 |
| **重复通知** | 同一事件多个提醒 | 体验差 |
| **时间不当** | 深夜发送非紧急消息 | 打扰休息 |
| **渠道分散** | 不知道消息从哪来 | 难以追溯 |

### Hermes的解决方案

```
Before (分散):                    After (统一):
├─ cron job 直接发送             ├─ 所有消息 → Hermes
├─ skill 直接调用 message()      ├─ Hermes路由 → 合适渠道
├─ 系统直接打印                   ├─ 优先级排序
└─ 各自为政                      └─ 合并去重 → 最佳时机发送
```

```
└─────────────────────────────────────────────────────────────┘
```

## 📋 消息分类与处理策略

### 优先级定义

| 优先级 | 定义 | 处理策略 | 示例 |
|:------:|:-----|:---------|:-----|
| **P0** | 紧急 | 立即推送 + 声音提醒 | 持仓爆仓风险、系统崩溃 |
| **P1** | 重要 | 立即推送 | 自诊断发现问题、进化提案 |
| **P2** | 一般 | 每小时汇总 | 定时任务完成、知识库更新 |
| **P3** | 低优先级 | 每日汇总 | 成功日志、统计报告 |

### 时间规则

```python
TIME_RULES = {
    "00:00-08:00": {  # 深夜/早晨
        "P0": "立即发送",      # 紧急不延迟
        "P1": "延迟到08:30",   # 重要改早晨
        "P2": "延迟到08:30",   # 一般改早晨
        "P3": "延迟到09:00",   # 低优先级改上午
    },
    "08:00-09:30": {  # 盘前
        "all": "立即发送",     # 交易时间前全部即时
    },
    "09:30-15:00": {  # 交易时间
        "P0": "立即发送",
        "P1": "立即发送",
        "P2": "每小时整点汇总",
        "P3": "延迟到15:30",
    },
    "15:00-23:00": {  # 盘后/晚间
        "all": "按优先级即时或汇总",
    }
}
```

## 🔄 工作流程

### 消息处理流程

```
消息产生
    ↓
发送至 Hermes
    ↓
【排队】进入消息队列
    ↓
【标记】打标签 (来源/优先级/时间敏感)
    ↓
【去重】检查重复 (相同内容合并)
    ↓
【时机】判断最佳发送时间
    ├── P0紧急 → 立即发送
    ├── P1重要 → 立即发送
    ├── P2一般 → 进入小时汇总
    └── P3低优先级 → 进入日汇总
    ↓
【发送】选择渠道推送
    ├── 飞书即时消息 (P0/P1)
    ├── 飞书卡片消息 (P2汇总)
    └── 每日简报 (P3汇总)
    ↓
【归档】记录到消息日志
```

### 智能合并示例

```
10:00 - SKILL_A: "完成扫描"
10:05 - SKILL_B: "更新完成"  
10:15 - SKILL_C: "归档成功"

→ Hermes合并为:
【10:30小时汇总】
- 3个SKILL完成日常维护
- 系统运行正常
```

## 🚀 使用方式

### Python API

```python
from hermes import Hermes

hermes = Hermes()

# 发送紧急消息 (P0)
hermes.send(
    content="持仓风险警报！",
    priority="P0",
    source="risk_management",
    channel="feishu"
)

# 发送一般通知 (P2)
hermes.send(
    content="知识库已更新",
    priority="P2",
    source="karpathy_wiki",
    # 自动进入小时汇总
)

# 批量发送 (自动合并)
hermes.batch_send([
    {"content": "任务1完成", "priority": "P3"},
    {"content": "任务2完成", "priority": "P3"},
    {"content": "任务3完成", "priority": "P3"},
])
# → 合并为: "3个任务已完成"
```

### 命令行

```bash
# 发送测试消息
python3 -m hermes send "测试消息" --priority P1

# 查看消息队列
python3 -m hermes queue

# 手动触发汇总发送
python3 -m hermes flush

# 查看消息统计
python3 -m hermes stats
```

## 📊 与现有系统集成

### 替换分散的消息调用

| 原调用 | 新调用 |
|:-------|:-------|
| `message.send()` 直接发送 | `hermes.send()` 统一路由 |
| `print()` 系统输出 | `hermes.log()` 归档记录 |
| `cron` 直接推送 | `cron → hermes` 时机优化 |

### 集成示例

```python
# 原方式 (分散)
from message import send
send(content="警报！", to="chief")

# 新方式 (统一)
from hermes import Hermes
hermes = Hermes()
hermes.send(
    content="警报！",
    priority="P0",
    source="trading_system"
)
# Hermes决定: 立即发送 + 飞书 + 声音提醒
```
