# Reflection-Optimizer SKILL

## 概述

基于Agentic Design Patterns的反思优化器，为A5L分析层提供自我纠错和质量提升能力。采用Producer-Critic双角色模型，通过迭代反思循环显著提升分析输出质量。

**设计模式来源**: Agentic Design Patterns Ch.4 Reflection (Gulli, 2025)
**架构归属**: Layer 3 Analysis - 每个分析师产出后强制执行
**核心能力**: 自我纠错、质量审查、偏见检测、迭代优化

## Producer-Critic双角色模型

### 角色定义

**Producer (生成者)**
- 职责: 生成初始分析输出
- 身份: 分析师角色 (如"行业研究员"、"价值投资者")
- 关注点: 全面性、深度、创造性
- 输出: 分析初稿

**Critic (批评者)**
- 职责: 审查Producer输出，识别问题
- 身份: 资深审查员 (如"首席投资官"、"风险总监")
- 关注点: 准确性、一致性、偏见、遗漏
- 输出: 结构化批评意见

## 六维度审查框架

### 维度1: 事实准确性 (Factual Accuracy)

**审查要点**:
- [ ] 数据来源是否可靠?
- [ ] 数值计算是否正确?
- [ ] 时间顺序是否准确?
- [ ] 引用信息是否可验证?

**示例批评**:
```
❌ 问题: "公司营收增长50%" - 未说明对比基准和期间
✓ 修正: "2025年Q1营收同比增长50%，达10亿元"
```

### 维度2: 逻辑一致性 (Logical Consistency)

**审查要点**:
- [ ] 论点之间是否有矛盾?
- [ ] 推理过程是否严密?
- [ ] 结论是否有充分支撑?
- [ ] 假设是否合理?

**示例批评**:
```
❌ 问题: 既说"行业处于上升期"又说"需求持续萎缩" - 逻辑矛盾
✓ 修正: 明确区分短期波动与长期趋势
```

### 维度3: 偏见检测 (Bias Detection)

**审查要点**:
- [ ] 是否存在确认偏误 (只找支持证据)?
- [ ] 是否存在过度乐观/悲观?
- [ ] 多空观点是否平衡?
- [ ] 是否存在锚定效应?

**示例批评**:
```
❌ 问题: 过度强调正面催化剂，对风险因素轻描淡写
✓ 修正: 增加"主要风险"章节，平衡呈现多空因素
```

### 维度4: 完整性检查 (Completeness Check)

**审查要点**:
- [ ] 是否遗漏关键信息?
- [ ] 是否回答了原始问题?
- [ ] 背景信息是否充分?
- [ ] 反面证据是否被考虑?

**示例批评**:
```
❌ 问题: 分析AI产业链但未提及GPU供应链瓶颈
✓ 修正: 补充"供应链风险"分析段落
```

### 维度5: 清晰度与表达 (Clarity & Expression)

**审查要点**:
- [ ] 语言是否清晰易懂?
- [ ] 专业术语是否解释?
- [ ] 结构是否层次分明?
- [ ] 重点是否突出?

**示例批评**:
```
❌ 问题: 长段落包含多个论点，难以阅读
✓ 修正: 拆分为要点列表，每个论点独立成段
```

### 维度6: 可操作性与价值 (Actionability & Value)

**审查要点**:
- [ ] 是否有明确结论?
- [ ] 是否提供可操作建议?
- [ ] 价值判断是否有依据?
- [ ] 时效性是否明确?

**示例批评**:
```
❌ 问题: "建议关注" - 过于笼统，无具体标准
✓ 修正: "建议关注：PE<20倍且QoQ增长>20%的标的"
```

## 集成到A5L分析管线

### UZI分析增强流程

```
原始流程:
输入 → UZI-Skill → 输出51维度评分 → 直接输出

增强后流程:
输入 → UZI-Skill (Producer)
          ↓
    [51维度评分初稿]
          ↓
    Reflection-Optimizer (Critic)
          ↓
    [六维度审查]
          ↓
    ├─ 通过 ──→ 输出最终报告
    │
    └─ 修订 ──→ UZI-Skill (Reviser)
                      ↓
              [优化后评分]
                      ↓
              Reflection-Optimizer (二次审查)
                      ↓
              ├─ 通过 ──→ 输出
              └─ 再修订 → (最多3轮)
```

### 产业链分析增强流程

```
原始流程:
输入 → 产业链分析 → 输出关系图 → 直接输出

增强后流程:
输入 → 产业链分析 (Producer)
          ↓
    [产业链图谱初稿]
          ↓
    Reflection-Optimizer (Critic)
          ├─ 检查: 实体识别完整性
          ├─ 检查: 关系逻辑一致性
          ├─ 检查: 与外部数据源一致性
          └─ 检查: 投资洞察深度
          ↓
    [修订意见]
          ↓
    产业链分析 (Reviser - 可选)
          ↓
    知识图谱系统 (自动更新)
          ↓
    输出最终图谱
```

```
原始流程:
输入 → 空方视角分析 → 输出风险清单

输入 → 空方视角分析 (Producer)
          ↓
    [风险清单初稿]
```

## 使用方式

### 触发指令

**独立使用**:
```
反思优化 [分析内容]
reflection [content]
审查分析 [analysis]
优化输出 [output]
```

**管线集成** (自动触发):
```
分析股票 000066 --with-reflection    → 自动在UZI后触发反思
深度分析 [标的] --quality-check      → 强制3轮反思优化
```

### 使用示例

**示例1: UZI评分反思**
```
用户: 分析中国长城

执行流程:
1. UZI-Skill生成51维度评分 (Producer)
2. Reflection-Optimizer启动 (Critic)
   - 检查: "成长性评分5/5" - 是否有过度乐观?
   - 检查: "估值评分3/5" - 对比历史PE是否合理?
   - 检查: "催化剂识别" - 是否遗漏最新政策?
3. 发现问题: 未考虑"减持风险"
4. UZI-Skill修订: 补充风险因素分析
5. 二次审查通过 → 输出最终报告
```

**示例2: 研报分析反思**
```
用户: 摄入这份MS研报

执行流程:
1. PDF提取内容 → 行业研究分析 (Producer)
2. Reflection-Optimizer审查:
   - 事实准确性: 核对关键数据与公开财报
   - 逻辑一致性: 观点之间是否有矛盾
   - 偏见检测: 是否存在投行利益冲突
   - 完整性: 是否遗漏重要风险因素
3. 发现问题: 对竞争格局分析过于乐观
4. 补充: 增加竞争风险章节
5. 输出: 带批注的完整分析
```

**示例3: 自检测试**
```
用户: 自检上次分析

执行流程:
1. 检索最近分析记录
2. Reflection-Optimizer回顾审查:
   - 对比实际走势验证准确性
   - 检查是否有事后诸葛亮偏见
   - 分析遗漏因素
3. 生成: 自检报告 + 改进建议
4. 更新: Memory Palace (避免重复错误)
```

## 质量门控标准

### 通过标准 (Pass Criteria)

必须同时满足:
- 六维度平均分 ≥ 4.0
- 无严重(高)级别问题
- 中等级别问题 ≤ 2个
- 偏见检测 ≥ 4分

### 修订标准 (Revise Criteria)

满足以下任一条件需修订:
- 平均分 3.0-4.0
- 存在1-2个严重问题
- 偏见检测 < 4分
- 完整性 < 3分

### 重审标准 (Redo Criteria)

满足以下任一条件需重审:
- 平均分 < 3.0
- 存在3+个严重问题
- 事实准确性 < 3分
- 逻辑一致性 < 3分
