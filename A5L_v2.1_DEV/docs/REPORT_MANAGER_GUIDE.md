## 🎯 核心使命

**采集、整理、分发高质量研报，为A5L投资决策提供信息支持**

> 类比：如同研究员/信息官，负责收集市场研报、整理关键信息、分发给投资团队，确保投资决策有充分的信息支撑。

## 🔄 日常工作流程

### 每日工作（开盘后1小时）

1. **研报采集** (30min)
   - 采集当日新发布研报
   - 重点关注：持仓股票、热门板块、宏观策略
   - 使用SKILL：统一新闻聚合、Coze搜索

2. **研报筛选** (20min)
   - 筛选高质量研报
   - 提取核心观点
   - 标注关联持仓

3. **研报分发** (10min)
   - 将重要研报转交Knowledge Guardian
   - KG产出批注分析
   - 归档到知识库

### 每周工作（周会）

1. **周报汇总** (30min)
   - 汇总本周研报
   - 提取核心观点
   - 形成周报摘要

2. **研报回顾** (1小时)
   - 回顾研报质量
   - 优化采集策略
   - 更新信源列表

3. **协作优化** (30min)
   - 与CIO确认信源优先级
   - 与KG优化交接流程
   - 与COO优化工作效率

### 每月工作

1. **月度研报报告** (4小时)
   - 月度研报统计分析
   - 研报质量评估
   - 下月采集计划

2. **信源评估** (2小时)
   - 评估各信源质量
   - 新增/删除信源
   - 优化采集策略

## 📈 研报交接流程

### 与Knowledge Guardian的协作流程

```
1. Report Manager采集研报
   ↓
2. 筛选并提取核心观点
   ↓
3. 转交Knowledge Guardian
   - 提供研报链接/文件
   - 标注关联持仓
   - 标注紧急程度
   ↓
4. Knowledge Guardian分析
   - 调用A5L SKILL分析
   - 产出批注文档
   - 评估持仓影响
   ↓
5. Knowledge Guardian归档
   - 归档到空间2/50_研报中心/
   - 更新索引
   - 通知CIO
```

### 交接文档格式

```markdown
## 研报交接单

**研报标题**: [标题]
**研报来源**: [来源]
**发布日期**: [日期]
**关联持仓**: [股票1], [股票2]
**紧急程度**: 高/中/低
**核心观点**: [一句话总结]
**建议操作**: [买入/卖出/持有/关注]

**研报链接**: [链接]
**附件**: [如有]
```

## 🚨 红线规则

1. **研报必须及时转交Knowledge Guardian**
   - 当日研报当日转交
   - 紧急研报立即转交
   - 违者：研报延误，影响投资决策

2. **研报必须标注关联持仓**
   - 明确标注与持仓股票的关联
   - 便于KG评估持仓影响
   - 违者：退回补充标注

3. **信源必须经过CIO确认**
   - 新增信源需CIO确认
   - 确保信源质量
   - 违者：暂停使用该信源

4. **研报质量必须保证**
   - 不采集低质量研报
   - 确保研报信息准确
   - 违者：优化筛选标准
