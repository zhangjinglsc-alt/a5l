# 双通道策略验证系统

> **设计目标**: 通过模拟盘+真实盘双通道，加速策略有效性验证和进化  
> **管理者**: L3 (策略层) + KG (知识管理)

---

## 双通道架构

```
┌─────────────────────────────────────────────────────────────┐
│                    策略有效性验证                            │
├─────────────────────────────────────────────────────────────┤
│  通道A: 模拟交易                    通道B: 真实交易         │
│  ├─ 自动化执行                      ├─ Chief手动执行        │
│  ├─ 高频数据积累                    ├─ 低频次高质量         │
│  ├─ 快速验证假设                    ├─ 验证实战可行性       │
│  └─ 统计显著性                      └─ 心理/执行因素        │
│                          ↓                                  │
│              数据融合 → 策略进化                            │
└─────────────────────────────────────────────────────────────┘
```

---

## 通道A: 模拟交易（自动化）

### 目标
- 快速积累大量交易样本
- 验证策略逻辑的有效性
- 统计回测 vs 实盘差异

### 当前系统

**A股模拟盘 (SignalArena)**
- 本金: ¥10,000,000
- 当前: ¥9,989,900 (-1.01%)
- 策略: 信创主线、价值投资

**美股模拟盘 (A5L内部)**
- 本金: $100,000
- 当前: 持有AAPL/NVDA/TSLA
- 策略: 科技龙头、AI算力

### 自动化流程
```
L3生成交易信号
    ↓
自动记录到模拟盘
    ↓
每日收盘自动同步
    ↓
计算盈亏、更新统计
    ↓
每周生成模拟盘报告
```

## 数据融合与策略进化

### 统计维度

| 维度 | 模拟盘 | 真实盘 | 融合价值 |
|------|--------|--------|----------|
| 胜率 | ✅ 大量样本 | ✅ 质量样本 | 加权平均 |
| 盈亏比 | ✅ 理想情况 | ✅ 含滑点/心理 | 校准系数 |
| 最大回撤 | ✅ 理论值 | ✅ 实际承受 | 风险评估 |
| 持仓时间 | ✅ 按规则 | ✅ 受情绪影响 | 行为修正 |

### 进化机制

**Step 1: 差异分析**
```
模拟盘胜率: 65%
真实盘胜率: 55%
差异: -10% （心理/执行损耗）
```

**Step 2: 原因归因**
- 过早止盈？
- 止损执行不坚决？
- 滑点成本？

**Step 3: 策略修正**
- 调整参数（考虑执行损耗）
- 优化入场/出场规则
- 强化心理训练

**Step 4: 再次验证**
- 模拟盘测试新参数
- 真实盘小仓位验证
- 闭环迭代
