## 核心流程

### 1. Goal 设定 → 人类职责
```
人类: "实现L3自主进化"
    ↓
Agent: 收到，正在分解目标...
    ↓
Agent: 分解完成，请确认任务树
```

### 2. Process 管理 → Agent职责
```
Agent: 当前进度 35%，下一步: 实现自修复
    ↓
Agent: 遇到阻塞: 需要您确认修复策略
    ↓
人类: 确认 / 调整 / 暂停
    ↓
Agent: 继续执行...
```

### 3. Result 验证 → 协作职责
```
Agent: 任务完成，自我评估: 85分
    ↓
Agent: 请验收成果
    ↓
人类: 验收通过 / 需要改进 / 重做
    ↓
Agent: 记录经验，固化模式
```

## 关键特性

### 📋 任务树 (Task Tree)
- **层级结构**: 目标 → 阶段 → 任务 → 子任务
- **状态流转**: 待开始 → 进行中 → 待验收 → 已完成
- **依赖关系**: 任务A完成才能开始任务B
- **时间预估**: 每个任务有预计耗时

### 📊 进度看板 (Progress Board)
- **整体进度**: 百分比 + 可视化进度条
- **燃尽图**: 剩余工作量趋势
- **阻塞标识**: 红色标注阻塞任务
- **风险预警**: 即将延期的任务

### 🔔 主动汇报 (Active Reporting)
- **每日简报**: 昨天完成、今天计划、阻塞风险
- **里程碑报告**: 阶段目标达成时汇报
- **异常告警**: 偏离计划时立即报告
- **验收请求**: 任务完成请求确认

### ✅ 验收机制 (Acceptance)
- **自评标准**: Agent先自我评分
- **成果展示**: 输出物 + 效果验证
- **改进建议**: 未达到100分时提出
- **经验记录**: 成功/失败模式萃取

## 运行模式

### 正常模式: 静默执行
- 按计划推进任务
- 每日自动生成简报（不打扰）
- 关键节点主动汇报

### 异常模式: 立即报告
- 任务延期风险
- 执行遇到阻塞
- 需要人类决策
- 系统出现错误

### 验收模式: 请求确认
- 阶段目标完成
- 任务达到里程碑
- 需要调整方向
